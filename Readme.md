# Real-Time Chat System (MongoDB + Availability)

## Features
- Real-time messaging using WebSocket
- MongoDB database
- Authentication (Login/Register)
- Availability-focused system (CAP Theorem)

## Tech Stack
- Java (Spring Boot)
- MongoDB
- WebSocket
- HTML/CSS/JS

## CAP Theorem
This system prioritizes Availability over Consistency.
Messages are always accepted and eventually become consistent.

## How to Run
1. Start MongoDB
2. Run backend
3. Open frontend/index.html
